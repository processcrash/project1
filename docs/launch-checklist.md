# Launch Checklist - CodeSentinel

## Pre-Launch

- [ ] Deploy backend to Railway
- [ ] Deploy frontend to Vercel
- [ ] Configure custom domain
- [ ] Set up Stripe products ($15 Pro, $49 Team)
- [ ] Add Stripe webhook endpoint
- [ ] Configure Sentry error tracking
- [ ] Test full user flow (register → create project → submit review)
- [ ] Verify email notifications work

## Launch Day

- [ ] Post to HackerNews (use Show HN format)
- [ ] Post to Dev.to
- [ ] Post Twitter thread
- [ ] Share on LinkedIn
- [ ] Post in relevant subreddits (r/programming, r/devops)
- [ ] Notify early signups from waitlist

## Post-Launch Week 1

- [ ] Monitor error rates (should be <1%)
- [ ] Check Stripe dashboard for trials
- [ ] Respond to all feedback
- [ ] Fix critical bugs within 24h
- [ ] Post update on social media

## Post-Launch Month 1

- [ ] Analyze conversion rates
- [ ] Calculate customer acquisition cost
- [ ] Identify top traffic sources
- [ ] Plan improvements based on feedback
- [ ] Consider Product Hunt launch

## Metrics to Track

- Daily Active Users (DAU)
- Reviews completed per day
- Sign-up conversion rate
- Trial to paid conversion rate
- Revenue (MRR)
- Customer feedback sentiment
- Average code quality score
- Most common issues found
